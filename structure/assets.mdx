# Assets

ZOODOM organizes assets into two locations: `src/assets/` for build-processed assets and `public/` for static files served as-is.

## Asset Directories

### src/assets/

Processed assets that are imported in code:

```
src/assets/
├── global.css       # Global styles and CSS variables (117 lines)
├── hero.png         # Hero section image
├── logo.svg         # ZOODOM logo vector graphic
└── texture-img.png  # Background texture for sections
```

**Characteristics:**
- Imported via frontmatter or component code
- Optimized at build time
- Content-hashed filenames in production
- Type-safe imports with TypeScript

### public/

Static assets served without processing:

```
public/
└── cover.png        # Open Graph social sharing image (669KB)
```

**Characteristics:**
- Served as-is from root URL
- No optimization or transformation
- Referenced by absolute path (e.g., `/cover.png`)
- Used for meta tags and external references

<Info>
Use `src/assets/` for images imported in components. Use `public/` for files referenced in meta tags or that need a predictable URL.
</Info>

## Importing Assets in Astro

### Image Imports

Astro processes image imports and returns an optimized object:

```astro
---
import logo from '../assets/logo.svg';
import HeroImg from '../assets/hero.png';
---

<img src={logo.src} alt="Logo" />
<img src={HeroImg.src} alt="Hero" />
```

Source: `src/pages/index.astro:3-4,14,60`

**Import object structure:**

```typescript
{
  src: '/assets/logo.abc123.svg',  // Optimized path with hash
  width: 200,                       // Image width (for raster)
  height: 100,                      // Image height (for raster)
  format: 'svg'                     // Image format
}
```

<Info>
SVG imports only have `src` and `format`. PNG/JPG imports include width and height for responsive images.
</Info>

### CSS Imports

CSS files are imported in the frontmatter:

```astro
---
import '../assets/global.css';
---
```

Source: `src/components/Head.astro:2`

Astro bundles the CSS and includes it in the page `<head>`.

### Background Image URLs

In CSS, use relative paths from the CSS file location:

```css
#animals {
    background-image: url('./texture-img.png');
}
```

Astro resolves the path relative to `global.css` and processes the image.

## Asset Optimization

### Automatic Optimization

Astro automatically optimizes assets imported in `src/assets/`:

- **Images** - Compressed and resized
- **CSS** - Minified and bundled
- **Filenames** - Content hashing for cache busting

### Build Output

After running `npm run build`, assets are copied to `dist/` with optimized names:

```
dist/
├── assets/
│   ├── logo.a1b2c3d4.svg
│   ├── hero.e5f6g7h8.png
│   └── global.i9j0k1l2.css
└── cover.png  (from public/, unchanged)
```

### Manual Optimization Tips

<AccordionGroup>
  <Accordion title="Optimize cover.png" icon="image">
    The social sharing image is 669KB. Recommended optimizations:
    
    - Resize to exactly 1200x630px (Open Graph standard)
    - Compress to 80-90% quality
    - Convert to WebP if browser support allows
    - Target size: under 200KB
    
    Tools: [TinyPNG](https://tinypng.com/), [Squoosh](https://squoosh.app/)
  </Accordion>
  
  <Accordion title="Use WebP for hero.png" icon="bolt">
    Convert PNG images to WebP format for smaller file sizes:
    
    ```astro
    ---
    import heroWebP from '../assets/hero.webp';
    import heroPNG from '../assets/hero.png';
    ---
    
    <picture>
      <source srcset={heroWebP.src} type="image/webp">
      <img src={heroPNG.src} alt="Hero">
    </picture>
    ```
  </Accordion>
  
  <Accordion title="Lazy load images" icon="clock">
    Add lazy loading for below-the-fold images:
    
    ```astro
    <img src={HeroImg.src} loading="lazy" alt="Hero">
    ```
    
    The browser will defer loading until the image is near the viewport.
  </Accordion>
</AccordionGroup>

## Google Fonts Integration

The Onest font is loaded via Google Fonts in the Head component:

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Onest:wght@100..900&display=swap" rel="stylesheet">
```

Source: `src/components/Head.astro:32-34`

**Font features:**
- Variable weights from 100 to 900
- `display=swap` for FOIT (Flash of Invisible Text) prevention
- Preconnect hints for faster loading
