# Multimedia Gallery

The multimedia gallery ("Galería") is a visual showcase that brings the zoo experience to life through high-quality photos and videos. It serves as both an engagement tool for potential visitors and a resource for current visitors to relive their experiences.

## Purpose

The gallery feature provides several key benefits:

- **Visual Appeal**: Attract visitors with stunning imagery of animals and exhibits
- **Social Sharing**: Encourage visitors to share their own photos and experiences
- **Marketing**: Provide promotional materials for media and partners
- **Education**: Illustrate animal behaviors, habitats, and conservation work
- **Engagement**: Keep the zoo top-of-mind between visits
- **User-Generated Content**: Build community through visitor photo submissions

## Gallery Types

### Main Gallery Categories

- **Animal Photos**: High-quality images of zoo residents
- **Visitor Experiences**: Photos of families enjoying the zoo
- **Behind the Scenes**: Staff working with animals, enrichment, training
- **Habitats & Exhibits**: Showcase of zoo facilities and environments
- **Events**: Special occasions, celebrations, educational programs
- **Conservation**: Field work, rehabilitation, breeding programs
- **Seasons**: How the zoo changes throughout the year

<Step title="Create Gallery Content Collection">

Set up a content collection for gallery items. Add to `src/content/config.ts`:

```typescript
import { defineCollection, z } from 'astro:content';

const galleryCollection = defineCollection({
  type: 'data',
  schema: ({ image }) => z.object({
    title: z.string(),
    description: z.string().optional(),
    image: image(),
    thumbnail: image().optional(),
    category: z.enum([
      'animals',
      'visitors',
      'behind-scenes',
      'habitats',
      'events',
      'conservation',
      'seasonal',
    ]),
    tags: z.array(z.string()).optional(),
    photographer: z.string().optional(),
    captureDate: z.date(),
    featured: z.boolean().default(false),
    species: z.string().optional(), // Link to species slug
    videoUrl: z.string().url().optional(), // For video items
  }),
});

export const collections = {
  'gallery': galleryCollection,
  // ... other collections
};
```

  // Filter functionality

      const filter = button.getAttribute('data-filter');

      galleryItems.forEach(item => {

      const title = button.getAttribute('data-title');

      lightboxTitle.textContent = title;

## Best Practices

### Image Specifications

- **Resolution**: Minimum 1920px wide for full-screen viewing
- **Format**: WebP for web, keep originals in high-quality JPEG/PNG
- **Aspect Ratio**: Maintain consistent ratios (4:3 or 16:9 recommended)
- **File Size**: Optimize to under 500KB per image
- **Thumbnails**: Generate 400x300px thumbnails for grid view

### Performance

- Lazy load images below the fold
- Use responsive images with `srcset`
- Implement progressive image loading (blur-up effect)
- Limit initial page load to 20-30 images
- Cache images aggressively

### Accessibility

- Provide descriptive alt text for all images
- Ensure keyboard navigation works in lightbox
- Add ARIA labels to filter buttons
- Support screen reader announcements for filters
- Maintain sufficient color contrast in overlays

### SEO

- Use descriptive filenames (e.g., `leon-africano-habitat.jpg`)
- Add alt text with relevant keywords
- Include image sitemaps
- Add structured data for ImageObject
- Optimize meta tags for social sharing (Open Graph)

<Tip>
  Add a "Download" button in the lightbox for high-resolution versions. This is useful for media, educators, and visitors who want wallpapers.
</Tip>
