# Event Calendar

The Event Calendar ("Calendario de eventos") is a priority feature that helps visitors discover and plan their zoo experience around scheduled activities, animal presentations, educational programs, and special events.

## Purpose

The event calendar provides several key benefits:

- **Visit Planning**: Help visitors plan their day to see specific shows and activities
- **Engagement**: Increase attendance at educational programs and presentations
- **Communication**: Promote special events like conservation days, holiday celebrations, or new exhibits
- **Organization**: Centralize all time-sensitive activities in one location

## Data Structure

Each event should include:

- **Title**: Event name (e.g., "Alimentación de Leones Marinos")
- **Date**: Event date or date range
- **Time**: Start and end time
- **Location**: Specific area within the zoo
- **Description**: Detailed information about the activity
- **Category**: Type of event (feeding, show, workshop, special event)
- **Age Group**: Target audience (all ages, children, adults)
- **Duration**: How long the event lasts
- **Capacity**: Maximum number of participants (if applicable)
- **Registration Required**: Whether advance signup is needed
- **Image**: Visual representation of the event

<Steps>

<Step title="Create Content Collection for Events">

Set up a content collection for events. Add to `src/content/config.ts`:

```typescript
import { defineCollection, z } from 'astro:content';

const eventsCollection = defineCollection({
  type: 'content',
  schema: ({ image }) => z.object({
    title: z.string(),
    date: z.date(),
    endDate: z.date().optional(),
    time: z.string(), // e.g., "10:00"
    endTime: z.string().optional(),
    location: z.string(),
    category: z.enum([
      'feeding',
      'show',
      'workshop',
      'tour',
      'special-event',
      'talk',
    ]),
    ageGroup: z.enum(['all', 'children', 'adults', 'families']),
    duration: z.string(), // e.g., "30 minutos"
    capacity: z.number().optional(),
    registrationRequired: z.boolean().default(false),
    registrationUrl: z.string().url().optional(),
    image: image().optional(),
    recurring: z.enum(['daily', 'weekly', 'monthly', 'none']).default('none'),
    daysOfWeek: z.array(z.number()).optional(), // 0-6 (Sunday-Saturday)
    featured: z.boolean().default(false),
  }),
});

export const collections = {
  'events': eventsCollection,
  // ... other collections
};
```

</Step>

import { getCollection } from 'astro:content';

const allEvents = await getCollection('events');

// Filter upcoming events
const today = new Date();
today.setHours(0, 0, 0, 0);

const upcomingEvents = allEvents
  .filter(event => event.data.date >= today)
  .sort((a, b) => a.data.date.getTime() - b.data.date.getTime());

// Group events by date
function groupByDate(events) {
  return events.reduce((acc, event) => {
    const dateKey = event.data.date.toISOString().split('T')[0];
    if (!acc[dateKey]) acc[dateKey] = [];
    acc[dateKey].push(event);
    return acc;
  }, {});
}

const eventsByDate = groupByDate(upcomingEvents);

const categoryIcons = {
  feeding: '🍽️',
  show: '🎭',
  workshop: '🎨',
  tour: '🗺️',

  talk: '💬',

  show: 'Espectáculo',
  workshop: 'Taller',
  tour: 'Visita Guiada',
  'special-event': 'Evento Especial',

  return new Intl.DateTimeFormat('es-ES', {
    weekday: 'long',
    year: 'numeric',

## Alternative: API-Based Calendar

For a more dynamic calendar with frequently changing events, consider using an API approach:

```typescript
// src/pages/api/events.json.ts
import type { APIRoute } from 'astro';
import { getCollection } from 'astro:content';

export const GET: APIRoute = async ({ url }) => {
  const events = await getCollection('events');
  const month = url.searchParams.get('month');
  const year = url.searchParams.get('year');

  let filteredEvents = events;

  if (month && year) {
    filteredEvents = events.filter(event => {
      const eventMonth = event.data.date.getMonth() + 1;
      const eventYear = event.data.date.getFullYear();
      return eventMonth === parseInt(month) && eventYear === parseInt(year);
    });
  }

  return new Response(
    JSON.stringify({
      events: filteredEvents.map(e => ({
        slug: e.slug,
        title: e.data.title,
        date: e.data.date,
        time: e.data.time,
        location: e.data.location,
        category: e.data.category,
      }))
    }),
    {
      headers: {
        'Content-Type': 'application/json'
      }
    }
  );
};
```

<Note>
  If you need to manage events through a CMS or allow staff to update events without deployments, consider integrating with a headless CMS like Sanity, Contentful, or Strapi instead of using Content Collections.
</Note>

## Best Practices

### User Experience

- **Calendar View**: Implement a visual calendar grid for monthly overview
- **Filter Options**: Allow filtering by category, date range, or age group
- **Today's Events**: Highlight events happening today prominently
- **Time Zones**: Display times in the zoo's local timezone

### Content Management

- **Recurring Events**: Handle daily/weekly events efficiently (feeding times)
- **Past Events**: Archive or hide past events automatically
- **Event Updates**: Plan for cancellations or time changes
- **Seasonal Events**: Mark special holiday or seasonal programming

### Mobile Optimization

- Use responsive card layouts for event listings
- Ensure touch-friendly buttons and links
- Consider a simplified day view for mobile devices
- Enable "Add to Calendar" functionality (iCal/Google Calendar)

### Performance

- Pre-render event pages at build time for fast loading
- Implement pagination for long event lists
- Cache API responses if using dynamic data
- Optimize event images for different screen sizes

<Tip>
  Add an "Add to Calendar" button that generates .ics files for each event. This allows visitors to save events directly to their personal calendars, increasing attendance.
</Tip>

## Integration with Other Features

- **Species Catalog**: Link feeding events to specific animal pages
- **Visit Planning**: Show today's events on the visit planning page
- **Newsletter**: Feature upcoming events in email campaigns
- **Homepage**: Display next 3 upcoming events on the main page

## Accessibility Considerations

- Use semantic HTML5 elements (`<time>`, `<article>`)
- Provide clear labels for all form fields (filters, search)
- Ensure keyboard navigation works for calendar controls
- Add ARIA labels for icon-only buttons
- Maintain sufficient color contrast for event categories
