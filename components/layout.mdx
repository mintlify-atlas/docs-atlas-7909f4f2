## Overview

The `Layout.astro` component serves as the base HTML layout for ZOODOM pages. It integrates Bootstrap 5.3.8 via CDN and provides the essential HTML structure with head and body sections.

<Info>
  This is the primary layout component that wraps all pages in the ZOODOM application.
</Info>

## Bootstrap Integration

ZOODOM uses **Bootstrap 5.3.8** loaded from jsDelivr CDN.

### Bootstrap CSS

Bootstrap styles are loaded in the `<head>` (`~/workspace/source/src/layouts/Layout.astro:9-14`):

```astro
<link
  href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/css/bootstrap.min.css"
  rel="stylesheet"
  integrity="sha384-sRIl4kxILFvY47J16cr9ZwB07vP4J8+LH7qKQnuqkuIAvNWLzeN8tE5YBujZqJLB"
  crossorigin="anonymous"
/>
```

<Note>
  The stylesheet includes SRI (Subresource Integrity) hash for security verification.
</Note>

## Head Component Integration

The layout imports and renders the `Head.astro` component (`~/workspace/source/src/layouts/Layout.astro:2,8`):

```astro
---
import Head from "../components/Head.astro";
---

<!doctype html>
<html lang="es">
  <head>
    <Head />
    <!-- Bootstrap CSS -->
  </head>
```

This provides:
- SEO meta tags
- Open Graph protocol support
- Page title and description
- Font loading (Onest from Google Fonts)
- Global CSS imports

<Tip>
  See the [Head Component](/components/head) documentation for details on passing custom title, description, and social image props.
</Tip>

## Slot Pattern

The layout uses Astro's slot pattern for content injection (`~/workspace/source/src/layouts/Layout.astro:17`):

```astro
<body>
  <slot />
  <!-- Bootstrap scripts -->
</body>
```

### How Slots Work

When you use the Layout component, child content is automatically inserted where `<slot />` appears:

```astro
---
import Layout from '../layouts/Layout.astro';
---

<Layout>
  <main>
    <h1>This content replaces the slot</h1>
    <p>Everything inside Layout tags appears in the body.</p>
  </main>
</Layout>
```

**Rendered output:**

```html
<!doctype html>
<html lang="es">
  <head><!-- Head component content --></head>
  <body>
    <main>
      <h1>This content replaces the slot</h1>
      <p>Everything inside Layout tags appears in the body.</p>
    </main>
    <!-- Bootstrap scripts -->
  </body>
</html>
```

## CDN vs. Local Bootstrap

### Current Implementation: CDN

**Advantages:**
- Fast delivery via jsDelivr's global CDN
- Browser caching across sites
- No build step required
- Automatic SRI security verification

**Disadvantages:**
- Requires internet connection
- External dependency
- Potential GDPR/privacy concerns
- No tree-shaking (full Bootstrap loaded)

### Alternative: Local Installation

To use local Bootstrap instead:

```bash
npm install bootstrap@5.3.8
```

```astro title="src/layouts/Layout.astro"
---
import 'bootstrap/dist/css/bootstrap.min.css';
import Head from "../components/Head.astro";
---

<!doctype html>
<html lang="es">
  <head>
    <Head />
  </head>
  <body>
    <slot />
  </body>
</html>

<script>
  import 'bootstrap/dist/js/bootstrap.bundle.min.js';
</script>
```

## Security: Subresource Integrity

All external resources include `integrity` attributes for SRI verification:

```astro
integrity="sha384-sRIl4kxILFvY47J16cr9ZwB07vP4J8+LH7qKQnuqkuIAvNWLzeN8tE5YBujZqJLB"
```

This ensures:
- Files haven't been tampered with
- CDN compromises don't inject malicious code
- Resources match expected content exactly

<Note>
  If you update Bootstrap versions, you must also update the integrity hashes. Get new hashes from [https://www.jsdelivr.com/](https://www.jsdelivr.com/).
</Note>

## Performance Optimization

### Script Loading Position

Scripts are loaded at the end of `<body>` rather than in `<head>`:

**Benefits:**
- Page content renders before JavaScript downloads
- Faster perceived page load
- JavaScript can safely access DOM elements

### CDN Performance

jsDelivr CDN provides:
- Global edge locations
- HTTP/2 and HTTP/3 support
- Automatic compression (gzip/brotli)
- Long-term browser caching

<Tip>
  Consider using `defer` or `async` attributes on script tags for even better performance, though this may require adjusting Bootstrap initialization.
</Tip>
