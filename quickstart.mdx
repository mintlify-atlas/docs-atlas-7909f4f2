# Quick Start

Get the ZOODOM website up and running on your local development environment in under 5 minutes.

## Prerequisites

Before starting, ensure you have:

- Node.js >= 22.12.0 installed
- Completed the [Installation](/installation) steps

## Development Workflow

<Steps>
  <Step title="Clone and Install">
    If you haven't already, clone the repository and install dependencies:

    ```bash
    git clone <repository-url>
    cd zoodom
    npm install
    ```
  </Step>

  <Step title="Start the Development Server">
    Launch the Astro development server using the npm script defined in package.json:9:

    <CodeGroup>
      ```bash npm
      npm run dev
      ```

      ```bash yarn
      yarn dev
      ```

      ```bash pnpm
      pnpm dev
      ```
    </CodeGroup>

    This runs the `astro dev` command which starts a local development server.

    You should see output similar to:

    ```bash
    astro v6.0.4 ready in 350 ms

    ┃ Local    http://localhost:4321/
    ┃ Network  use --host to expose
    ```
  </Step>

  <Step title="Open in Browser">
    Navigate to `http://localhost:4321/` in your web browser.

    You'll see the ZOODOM homepage with:

    - **Responsive Navigation Bar**: Logo with navigation links (Inicio, Especies, Galeria, Blog, Contacto)
    - **Hero Section**: Large heading "Bienvenido a nuestro zoológico" (src/pages/index.astro:55)
    - **Call-to-Action Button**: "Reservar ahora" button styled with brand green (src/pages/index.astro:57)
    - **Hero Image**: Positioned on the right side on tablet/desktop (hidden on mobile)
  </Step>

  <Step title="Start Developing">
    The development server features hot module replacement (HMR). Any changes you make to files will automatically reload in the browser.

    Try editing `src/pages/index.astro` to see live updates!
  </Step>
</Steps>

## Available Scripts

ZOODOM includes the following npm scripts defined in package.json:8-13:

### Development

```bash
npm run dev
```

Starts the Astro development server at `http://localhost:4321` with hot module replacement.

### Build for Production

```bash
npm run build
```

Builds the site for production to the `./dist/` directory. This command:
- Compiles all `.astro` files to static HTML
- Optimizes assets and images
- Generates the production-ready site

### Preview Production Build

```bash
npm run preview
```

Serves the production build locally for testing before deployment. Run this after `npm run build` to verify the production site works correctly.

### Astro CLI

```bash
npm run astro
```

Directly access Astro CLI commands. Examples:

```bash
npm run astro add <integration>  # Add integrations
npm run astro check              # Type-check your project
npm run astro -- --help          # Get help
```

## Production Deployment

The site is configured for deployment at **https://zoodomi.netlify.app** (astro.config.mjs:6).

<Steps>
  <Step title="Build the Site">
    ```bash
    npm run build
    ```

    This generates static HTML in the `./dist/` directory.
  </Step>

  <Step title="Test Production Build">
    ```bash
    npm run preview
    ```

    Verify everything works as expected before deploying.
  </Step>

  <Step title="Deploy to Netlify">
    The `dist/` folder can be deployed to Netlify or any static hosting provider.

    The site URL is configured in `astro.config.mjs` for proper canonical URLs and sitemap generation.
  </Step>
</Steps>

<Note>
  The production site uses the canonical URL `https://zoodomi.netlify.app` which is referenced in Open Graph tags (src/components/Head.astro:13,39) for proper social media sharing.
</Note>
