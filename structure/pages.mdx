# Pages

Astro uses file-based routing where each `.astro`, `.md`, or `.mdx` file in the `src/pages/` directory automatically becomes a route on your site.

## File-Based Routing

### How It Works

The file path determines the URL:

| File Path | URL Route |
|-----------|----------|
| `src/pages/index.astro` | `/` |
| `src/pages/about.astro` | `/about` |
| `src/pages/blog/post.astro` | `/blog/post` |
| `src/pages/species/[id].astro` | `/species/:id` (dynamic) |

<Info>
Currently, ZOODOM has only one page: `index.astro` which serves as the homepage at `/`.
</Info>

### Route Generation

Astro automatically:
- Discovers all files in `src/pages/`
- Creates routes based on file names
- Pre-renders pages at build time (by default)
- Generates static HTML files in `dist/`

## index.astro - Homepage

The homepage is the main entry point for the ZOODOM website.

**Location:** `src/pages/index.astro`  
**Route:** `/`  
**Lines:** 96

### Page Structure

The page is organized into distinct sections:

1. **Imports** (frontmatter)
2. **Navigation** (navbar)
3. **Header** (hero section)
4. **Content sections** (animals, future content)

### Frontmatter Imports

```astro
---
import Layout from '../layouts/Layout.astro';
import logo from'../assets/logo.svg';
import HeroImg from '../assets/hero.png';
---
```

Source: `src/pages/index.astro:1-5`

**What's imported:**
- `Layout` - The main layout wrapper with Bootstrap
- `logo` - SVG logo as an optimized asset object
- `HeroImg` - Hero section PNG image

<Note>
Assets imported in the frontmatter are processed by Astro's build system. Astro returns an object with a `src` property containing the optimized image path.
</Note>

### Layout Wrapper

```astro
<Layout>
	<main>
		<!-- All page content -->
	</main>
</Layout>
```

Source: `src/pages/index.astro:7-94`

The entire page is wrapped in the `<Layout>` component and contained within a `<main>` element for semantic HTML.

## Image Asset Usage

### Logo Import Pattern

```astro
---
import logo from'../assets/logo.svg';
---

<img src={logo.src} />
```

Astro processes the SVG and returns an object. Access the path via `.src` property.

### PNG Image Import Pattern

```astro
---
import HeroImg from '../assets/hero.png';
---

<img src={HeroImg.src} />
```

Same pattern for raster images - Astro optimizes them at build time.

<Tip>
Always use the `.src` property when referencing imported images in `<img>` tags or CSS.
</Tip>

## Page Performance

### Static Generation

By default, Astro pre-renders this page at build time:

1. Runs the frontmatter code
2. Imports and optimizes assets
3. Renders the HTML template
4. Generates static `index.html` in `dist/`

### Asset Optimization

Imported images are automatically:
- Optimized for web
- Given cache-friendly filenames
- Served from the build output

### JavaScript

The only JavaScript on the page comes from Bootstrap CDN (loaded in Layout.astro):
- Popper.js for tooltips/dropdowns
- Bootstrap.js for interactive components
- Bootstrap bundle

## Creating Additional Pages

To add more pages to ZOODOM:

### 1. Create a new .astro file

```bash
touch src/pages/especies.astro
```

### 2. Use the same layout pattern

```astro
---
import Layout from '../layouts/Layout.astro';
---

<Layout>
	<main>
		<h1>Nuestras Especies</h1>
		<!-- Content -->
	</main>
</Layout>
```

### 3. Update navigation links

In `index.astro`, update the href:

```astro
<a class="nav-link" href="/especies">Especies</a>
```

### 4. Build and deploy

The new page will automatically be generated at build time.

## Dynamic Routes

For dynamic pages (e.g., individual species pages), use bracket notation:

**File:** `src/pages/especies/[id].astro`  
**Routes:** `/especies/leon`, `/especies/tigre`, etc.

```astro
---
export function getStaticPaths() {
  return [
    { params: { id: 'leon' } },
    { params: { id: 'tigre' } },
    { params: { id: 'elefante' } },
  ];
}

const { id } = Astro.params;
---

<Layout>
	<h1>Especie: {id}</h1>
</Layout>
```

<Info>
Dynamic routes require `getStaticPaths()` to tell Astro which pages to generate at build time.
</Info>
